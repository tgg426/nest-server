## 💬 用户发送消息接口：`sendmessage`

前端用户发送对话消息给大模型，可附带上传的文件信息。

### 请求方式：

`POST`

### 接口地址：

`/sendmessage`

### 请求参数：

| 参数名           | 类型            | 是否必填 | 说明                                     |
| ---------------- | --------------- | -------- | ---------------------------------------- |
| content          | string          | 是       | 用户发送的文本消息内容                   |
| uploadFileList   | UploadFileDto[] | 否       | 上传的文件列表（每项为单个文件对象）     |
| sessionId        | string          | 是       | 当前对话的唯一标识符（MongoDB ObjectId） |
| isKnowledgeBased | boolean         | 否       | 是否基于知识库进行回答，默认 false       |

### UploadFileDto 结构：

上传文件对象结构如下：

| 字段名   | 类型   | 是否必填 | 说明              |
| -------- | ------ | -------- | ----------------- |
| fileName | string | 是       | 文件名            |
| fileSize | string | 是       | 文件大小          |
| fileType | string | 是       | 文件类型          |
| docId    | string | 是       | 文件对应的文档 ID |

### 请求示例：

```json
{
  "content": "请帮我总结一下这份文件的重点内容。",
  "uploadFileList": [
    {
      "fileName": "example.pdf",
      "fileSize": "102400",
      "fileType": "pdf",
      "docId": "123456rfghhj"
    }
  ],
  "sessionId": "1234dfghjjkliou",
  "isKnowledgeBased": true
}
```
